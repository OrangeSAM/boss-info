# JD采集助手 - 产品路线图

## 现状

Chrome 扩展，核心功能：
- BOSS直聘岗位列表采集（多公司）
- AI 分析岗位 JD（多 Provider）
- Markdown 报告生成与导出
- 深色/浅色主题切换

数据全部存储在用户本地（chrome.storage），无后端。

---

## 一、产品完善（现在就能做）

### 新用户引导
- 第一次打开时的 onboarding 流程
- 引导用户配置 API Provider
- 引导用户到 BOSS直聘采集第一个公司

### 错误处理
- API 配置错误时的友好提示（地址拼写、Key 无效等）
- 网络异常时的重试引导
- 采集失败时的排查建议

### 数据安全提示
- 明确告知用户数据只存本地，不上传
- 隐私声明页面或弹窗

### 快捷操作
- Popup 显示最近采集的公司，一键跳转分析
- 历史分析记录快速访问

---

## 二、获客层面

### Chrome Web Store 优化
- 高质量截图（亮色 + 暗色）
- 精准的描述和关键词 SEO
- 定期更新版本保持排名

### 落地页
- 一个简单的产品介绍页
- 核心卖点展示：采集 → 分析 → 报告，一站式
- 下载引导

### 内容营销
- V2EX、掘金、小红书发使用体验帖
- 制作 demo 视频
- 技术博客分享开发过程（吸引开发者用户）

### 口碑传播
- 分析报告页加水印/分享按钮，带产品链接
- 邀请好友机制

---

## 三、商业化层面

### 定价模型
- 免费：每月 N 次分析
- 付费：解锁无限次分析
- 可选增值：多设备同步、历史记录云端存储

### 账号体系
- 需要后端支持用户登录
- 配额管理（免费/付费）
- 用户数据关联

### 支付接入
- 微信支付 / 支付宝
- 国际市场可加 Stripe

### API 代理（可选）
- 面向非技术用户，内置 AI API
- 用户不需要自己配置 Key
- 按用量计费

---

## 四、关键 gap

**最大瓶颈：没有后端。**

当前所有数据都在用户本地，无法实现：
- 付费墙（配额限制）
- 用户统计（DAU、留存）
- 云同步（多设备）
- 使用分析（哪些功能被高频使用）

**建议优先级：**
1. 加轻量后端（用户系统 + 配额）
2. Chrome Web Store 上架优化
3. 落地页 + 内容营销
4. 付费功能上线
